## 🟡 Partially Implemented

### Framework Scoring — Depth Gaps

**BSS Formula (Blueprint spec vs actual)**
```
Blueprint:  BSS = (0.25×Revenue) + (0.20×Margin) + (0.20×Debt) + (0.15×Moat) + (0.10×Management) + (0.10×Industry)
Actual:     scoring/allScores.js uses simplified ratio-to-score mapping
```
- ❌ **Moat score** — Blueprint calls for qualitative→quantitative scoring rubric. Currently not scored separately.
- ❌ **ROIC consistency tracker** — 5-yr rolling ROIC with trend classification not implemented
- ❌ **BSS deterioration alert** — QoQ drop >1.5 points rule exists but BSS history not tracked

**IOS Formula (Blueprint vs actual)**
```
Blueprint:  IOS = (BSS×0.30) + ((10-VRS)×0.22) + (MOS×0.18) + (STI_adj×0.18) + (AltData×0.12)
            STI_adj = 10 - |STI-50|/5  (neutral sentiment rewarded)
Actual:     composite = (BSS×0.30) + ((10-VRS)×0.20) + (MOS×0.20) + (STI_adj×0.18) + (AltData×0.12)
```
- ⚠️ Weights slightly off from spec (VRS weight 0.22→0.20, MOS 0.18→0.20)
- ⚠️ STI_adj formula — check if implemented correctly in `allScores.js`

**STI Score**
- Blueprint: VIX + Put/Call ratio + A/D ratio + NLP sentiment velocity → 0–100 scale
- Actual: Simplified score derived from promoter pledge + FII flow proxy → 0–10 scale
- ❌ No Put/Call ratio data
- ❌ No A/D (Advance/Decline) ratio
- ⚠️ Scale mismatch: Blueprint uses 0–100, app uses 0–10 (conversion: app_score × 10)

**MOS — Market Opportunity Score**
- Blueprint: Sector rotation stage + M2 growth + RBI repo trend + FII/DII flows + institutional participation
- Actual: RBI rate + FII flow + VIX from macro table ✅ but:
- ❌ No sector rotation classification engine (rolling 4-week relative strength)
- ❌ No M2 money supply tracking

## 🎯 Priority Gap List — What to Build Next

Ranked by **impact vs effort** for a solo developer:

### Tier 1 — High Impact, Moderate Effort
1. **Real Alt Data Ingestion** — Connect `data.gov.in` FASTag + E-Way Bill APIs (both free/public). This is your biggest competitive edge and the Framework 5 score is currently meaningless with seeded data.
2. **LLM Integration (Gemini/Claude API)** — The thesis generator, auto-review drafter, and weekly brief are all scaffolded but use hardcoded text. One real API key transforms 4 modules.
3. **IOS Weight Correction** — Fix weights to match blueprint spec exactly: VRS weight 0.22, MOS 0.18. Small but technically important.
4. **Brier Score + Calibration Curve** — The calibration section in Scenarios is partially built. Add `brier_score_loss` calculation and chart. This is 1–2 days of work.
5. **TipTap Editor in Decision Notes** — Replace `<textarea>` with rich text. Blueprint specifically calls this out.

### Tier 2 — High Impact, Higher Effort
6. **Scale to 200+ Stocks** — Currently 20 stocks. Blueprint spec is 500+. Add bulk seed from a screener export (Screener.in allows CSV exports).
7. **Real-time Cron for Alt Data** — Create `server/jobs/altDataUpdater.js` similar to `priceUpdater.js`, ingesting FASTag/E-Way data on a weekly schedule.
8. **Quarterly Review Auto-Trigger** — Decision notes older than 3 months should auto-flag for review. Node-cron can scan and push a notification.
9. **Moat Scoring Module** — Currently missing entirely. Add a manual qualitative input (switching cost, network effect, pricing power, ROIC) converted to BSS sub-score.
10. **TypeScript Migration** — Blueprint specifies TypeScript. Add types incrementally — especially for API response shapes.

### Tier 3 — Phase 4+ (Future)
11. Phase 5 Portfolio Optimizer (PyPortfolioOpt)
12. Phase 4 FinBERT NLP pipeline
13. Phase 6 Backtesting with Backtrader
14. Phase 7 Auth + SaaS infrastructure
