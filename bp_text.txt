PART 2 — Decision Notes System Every investment action must be documented in a structured Decision Note before execution. This creates an auditable journal enabling post-facto bias analysis, strategy refinement, and AI learning-loop training. 2.1  Pre-Trade Documentation Fields Field What to Document Entry Reason Detailed thesis referencing BSS, VRS, MOS, STI scores. Why this stock, why now? Minimum 3 sentences. Risk Factors Top 3–5 risks: earnings miss, regulatory headwind, promoter pledge, macro deterioration. Explicit, not vague. Time Horizon Short-term trade (&lt; 3 months) / Medium (3–12 months) / Long-term (&gt; 12 months). Determines review schedule. 3-Scenario Model Bull / Base / Bear case with price targets, probability %, and timeframe. Feeds Expected Value calculation. Expected Value EV = (P_bull × R_bull) + (P_base × R_base) + (P_bear × R_bear). Only enter if EV meaningfully positive. Invalidation Criteria &apos;I will exit if X occurs.&apos; Specific, not vague. E.g.: EPS miss &gt; 15%, stock drops 20% without market decline. Position Sizing % of portfolio, absolute INR, shares. Linked to IOS conviction score via Kelly Criterion sizing. Narrative Check What is the dominant market narrative driving current valuation? What single event destroys it? 2.2  Post-Trade Review (3 &amp; 6 Month Triggers) Review Field What to Analyze Outcome vs Expectation Actual return vs Bull/Base/Bear case. Which scenario played out? Why? Probability Calibration Were your probability estimates accurate? Track calibration score across 50+ trades. Bias Identification Tag cognitive biases: Confirmation, Anchoring, Recency, Overconfidence, Loss aversion. Data Quality Review Did BSS/VRS/STI signals predict correctly? Which framework sub-factor failed? Narrative Outcome Did the narrative hold or break? What was the breaking event? Was it in your risk list? Strategy Correction What rule changes based on this outcome? Update System Logic in Part 3. Pattern Library Tag Classify trade into reusable pattern: &apos;Contrarian reversal&apos;, &apos;Sector rotation play&apos;, &apos;Narrative break exit&apos;. 2.3  Technical Implementation USE EXISTING: PostgreSQL table &apos;decision_notes&apos; — all fields structured, linked to company_id, framework score snapshot. PART 3 — System Logic &amp; AI Reasoning Engine The core intelligence layer converts structured framework scores into actionable signals via rule-based logic, probabilistic scoring, and ML models. Everything here must be explainable — not a black box. 3.1  Rule-Based Decision Engine Condition Action Rationale IF rates rising + MOS ≤ 4 Reduce high-VRS growth exposure Tighten valuation in rate-rising regime IF VIX &gt; 25 + STI &lt; 20 + BSS &gt; 7 Flag contrarian buy candidate Quality stock in fear zone = asymmetric opportunity IF BSS &gt; 7 + MOS &gt; 7 + STI 40–60 Bullish bias, increase position Multiple confirmations = high conviction IF STI &gt; 80 + VRS &gt; 7 Flag partial profit booking Greed + expensive = distribution risk IF BSS drops &gt; 1.5 QoQ + VRS &gt; 5 Initiate exit review process Fundamental deterioration at high price IF FII selling &gt; 3 days + MOS ≤ 3 Shift defensive + raise cash Institutional exit = weak market signal IF Narrative Risk Score &gt; 7 Reduce position, tighten stop High narrative break risk = asymmetric downside IF Alt Data signal diverges from BSS Flag for deep-dive review Leading indicator conflict = uncertainty 3.2  Investment Opportunity Score (IOS) IOS = (BSS × 0.30) + ((10 − VRS) × 0.22) + (MOS × 0.18) + (STI_adj × 0.18) + (AltData × 0.12) STI_adj = 10 − |STI − 50| ÷ 5   (neutral sentiment rewarded; extremes discounted for IOS but tracked separately) AltData = Alternative Data Signal score (Framework 5) — added as 12% weight IOS ≥ 7.5 → Strong Buy    |    IOS 5.0–7.4 → Watchlist    |    IOS &lt; 5.0 → Avoid 3.3  Machine Learning Layers USE EXISTING: XGBoost (GitHub: dmlc/xgboost) — return prediction model. When that narrative breaks, price collapses fast regardless of fundamentals. No Indian retail tool tracks this. This is your edge. NRS Components: ► Narrative Identification: What is the single story the market tells about this stock? (entered manually per decision note) ► Consensus Level: How widely held is this narrative? (proxy: analyst coverage uniformity + media mention frequency) ► Narrative Sensitivity: What single event would destroy it? (earnings miss, regulatory reversal, competitor entry, etc.) ► NRS = (Consensus Score × 0.5) + (Sensitivity Score × 0.5)  |  Range 0–10 ► NRS &gt; 7: High narrative break risk — reduce position, tighten stop loss ► Monitoring: NLP scan of earnings call transcripts for narrative-cracking language patterns USE EXISTING: sec-edgar-downloader (GitHub: jadchaar/sec-edgar-downloader) — earnings call transcripts for US stocks. NEW IDEA #7 — Outcome Probability Scoring System Your current system produces a composite IOS score and a Buy/Hold/Avoid signal. That is binary thinking. Serious investors think in probability-weighted outcomes. This system forces honest thinking about downside and creates a trackable calibration record over time. The Three-Scenario Model Component Description Bull Case Best realistic outcome (not fantasy). Define: price target, timeframe, conditions required, probability (sum of all three = 100%). Base Case Most likely outcome. The scenario you weight most heavily. Detailed: what needs to go right, what can go slightly wrong. Bear Case Realistic downside — not catastrophe but genuine risk scenario. What happens if main thesis is partially wrong? Expected Value EV = (P_bull × R_bull) + (P_base × R_base) + (P_bear × R_bear). Only enter if EV is meaningfully positive. Minimum EV Threshold Configurable by user. Suggested: EV &gt; 15% over your time horizon minimum. Reject trades below this threshold. Calibration Score Over 50+ trades: Are your 70% confidence calls right 70% of the time? Brier score + calibration curve. This is your meta-skill. Calibration Scoring — The Most Valuable Long-Term Metric Brier Score = (1/N) × Σ (probability_forecast − actual_outcome)² Perfect calibration: Brier Score = 0. Random forecasting: Brier Score = 0.25. Track separately for Bull/Base/Bear probability estimates. Display as calibration curve: if you say 70%, do outcomes happen 70% of the time? Over 50+ trades this becomes your most valuable personal data asset — shows exactly where you are overconfident vs underconfident. Feeds Learning Engine: recalibrate IOS weights based on which scenarios actually materialized. USE EXISTING: scikit-learn brier_score_loss and calibration_curve — standard functions, 2 lines of code. USE EXISTING: Plotly reliability diagram — calibration curve visualization. Standard chart type. USE EXISTING: PostgreSQL — add columns to decision_notes table: bull_p, base_p, bear_p, bull_target, base_target, bear_target, ev_calculated. BUILD NEW: EV calculator widget in Decision Notes UI — 3-scenario input with real-time EV display and minimum threshold gate.
